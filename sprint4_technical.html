Got blocked on a simple problem

The problem I faced was with using function parameters in dot notation.

What was the problem?

I wanted to be able to access an object called gradebook which has child values of students Joseph, Susan, Wiremu, Elizabeth each of them had a testScores value which as an ordered list of their scores. so gradebook.elizabeth.testScores would equal [100, 90, 95, 85]

I had a function took 3 parameters addScore(book, student, score), book student and score the book parameter was gradebook, the student value was Susan, the score was 80. I knew I could use the method push to add score param to the array gradebook.Suscan.testScore, I want to be able to do this.

book.student.testScore.push(score) but this was throwing an error, testScore undefined.
console.log(gradebook.Susan.testsScore) would return [100, 90, 95, 85]
console.log(book.Susan.testsScore) would return [100, 90, 95, 85]
gradebook.Suscan.testScore.push(score) Worked ! but I didn't want to hardcode gradebook.

This confused me to the point of frustration for a while I couldn't figure out what was going wrong, I started googling and read about someone having some similar problems on stack overflow that suggested using bracket notation might be the answer, after a little research on youtube

I found this would do the job done.
book[student].testScores.push(score)

When you know what terms to google its a powerful tool, but sometimes finding the right question to ask is the hardest part. If my frist set or two of search terms do not return helpful results it's good to take a step back and think am I asking the right question, is there another way to solve this ? This is also a good point to consult the duck! explain actually what your trying to do, while writing the steps down hopefully explaining it will give you a lead, or turn your notes into pseudocode. Personally I get more out of a good youtube video on a subject than reading people's blog posts or tech articles, a solid 5-10 minute video explaining a method or feature is gold to me. I need to practice being mindful of when to try another technique and not get too focused on one approach.

Solved a problem in an elegant way.

Also during the gradebook challenge we had to get the average of test scores stored in array, I knew I could loop through the array and add each value together, I knew there must be a better or more interesting way of doing this and recalled that in other languages you could reduce an array of integers to a single value. after a little trial and error and youtube video about the reduce method I was able to have an elegant way of adding all the values together then dividing them by the number of items in the array with the length method.

function average(arr){
sum = arr.reduce((acc, val) => acc + val);
return(sum / arr.length)
}

I learnt about arrow function syntax away of writing a function inline and the reduce method was more complex than I initially thought as well, I didn't expect to have to pass an accumulator and currentValue to the reduce method.
It felt rewarding to find a more elegant way to write this and I enjoyed learning about new techniques
