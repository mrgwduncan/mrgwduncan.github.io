best practices associated with using classes vs. ids

Both classes and ID allow attributes to be applied to an element of html. for example if we had a class called blogpost and in the css we stated background-color:black then we had a div element with the class="blogpost" will have a black background. This is the same for id, if we define the id firstpost to have a black ground in the css then a div element with id="firstpost" will have a black background.

However a elemental wether it be a div, p, span, h1 can only have 1 ID but mutliple classes and a ID should be unique where a class can but reused on multiple elements of a page. a good example could be the class blogpost would be apporiate to style all your blogposts on your site, defining attributes like font, height, width, background color and postion, each blogpost would have the class="blogpost", you might have a second class that changes the colors of the most recent blog post to highlight its importance class="blogpost recent". An apporiate use of the id tag in this example would be to have each post have a sequential number in the id like id="post1" for the first post and id="post2" for the second post and so fourth.

Once you start using javascript to controll your page ID really start to shine with functions like getElementById, say you wanted to have a button to hide all your posts expect the one the person is reading, you might have javascript change the display attribute to none on all the blog posts here some pusdo code for an example:
